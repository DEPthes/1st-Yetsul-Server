# 프로젝트 규칙

- 문서화 하기
- 코드, 커밋 규칙 준수하기
- 모든 코드와 커밋은 따로 브런치 생성 후 머지하기
- 커밋 메시지는 "[Type] message" 형식으로 작성하기
  | **Type** | **Meaning** |
  | -------- | ------|
  | feat | 새로운 기능 |
  | fix | 버그 수정 |
  | build | 빌드 관련 파일 수정 |
  | chore | 그 외 자잘한 수정 |
  | style | 코드 스타일 또는 포맷 |
  | docs | 문서 수정 |
  | refactor| 코드 리팩토링 |
  | !BREAKING CHANGE | 단절적 변경 |
